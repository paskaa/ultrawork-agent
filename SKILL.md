```
┌─────────────────────────────────────────────────────────────┐
│                     1. INTENT ANALYSIS                       │
│  ZhugeLiang 分析用户意图，JiangWei 分解复杂任务               │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     2. TASK DISPATCH                         │
│  根据任务类别，指派对应主将 + 部将组合                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                     3. PARALLEL EXECUTION                    │
│  主将统筹，部将并行执行子任务                                 │
│                                                             │
│  ┌─────────┐                                                │
│  │ 主将     │ ──协调──> 部将A (子任务1)                      │
│  │         │ ──协调──> 部将B (子任务2)                      │
│  └─────────┘                                                │

                           ▼

```

### 状态图标

| 图标 | 含义 |
|------|------|
| `[IDLE]` | 等待中 |
| `[RUN]` | 执行中 |
| `[OK]` | 已完成 |
| `[FAIL]` | 失败 |
| `[SKIP]` | 跳过 |

### 进度条

- `#` = 已完成
- `-` = 剩余
- 示例: `[######------] 50%`

## 部将调用规则

### 规则1: 部将必须由主将调用

```
❌ 错误: 用户直接调用 GaoShun
✅ 正确: ZhaoYun 调用 GaoShun 执行子任务
```

### 规则2: 部将可并行执行

```
ZhaoYun 统筹:
├── GaoShun: 处理核心逻辑
└── ChenDao: 处理边缘情况
    (并行执行，提高效率)
```

### 规则3: 部将结果汇总到主将

```
GaoShun 完成 --> 汇报 ZhaoYun
ChenDao 完成 --> 汇报 ZhaoYun
ZhaoYun 汇总 --> 汇报 ZhugeLiang
```
